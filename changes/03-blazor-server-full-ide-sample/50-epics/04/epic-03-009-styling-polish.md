# Epic 03-009: Styling and Visual Polish

**Phase**: 04 - Polish
**Status**: Ready
**Priority**: Medium

## Overview

Refine visual styling to match OpenCode design and ensure consistent look and feel.

## Stories / Tasks

### Design Audit

- [ ] **Task 1**: Compare with OpenCode reference
  - Screenshot OpenCode web UI
  - Document visual differences
  - Create list of refinements needed

- [ ] **Task 2**: Review color palette
  - Ensure dark theme colors match
  - Verify accent colors
  - Check contrast ratios for accessibility

### Component Styling

- [ ] **Task 3**: Refine file tree styling
  - Icons and colors
  - Selection states
  - Hover effects
  - Spacing and indentation

- [ ] **Task 4**: Refine chat styling
  - Message bubble appearance
  - Avatar/icon styling
  - Code block appearance
  - Streaming indicator

- [ ] **Task 5**: Refine diff viewer styling
  - Line colors and backgrounds
  - Gutter appearance
  - Syntax highlighting colors

- [ ] **Task 6**: Refine terminal styling
  - Font and colors
  - Scrollbar appearance
  - Output type differentiation

### Layout Polish

- [ ] **Task 7**: Refine pane borders and dividers
  - Resize handle appearance
  - Panel header styling
  - Tab bar appearance

- [ ] **Task 8**: Add visual feedback
  - Loading states
  - Error states
  - Success states
  - Transitions and animations

### Typography

- [ ] **Task 9**: Review typography
  - Font sizes and weights
  - Line heights and spacing
  - Code font (monospace)

## Acceptance Criteria

- [ ] Visual appearance closely matches OpenCode
- [ ] All components have consistent styling
- [ ] Visual feedback for all states
- [ ] Proper contrast for accessibility
- [ ] No visual glitches or misalignments

## Dependencies

- All Phase 03 epics must be complete
